# HR Database Management System

## Project Overview

The HR Database Management System is a SQL-based project designed to manage employee, department, job, location, and dependent information using a structured relational database. The project demonstrates database design, SQL querying, and HR data analysis to support business decision-making.

---

## Project Objective

* Design a relational HR database.
* Store and manage employee information efficiently.
* Analyze workforce, salary, and department data.
* Generate business insights using SQL queries.

---

## Database Tables

The database consists of the following tables:

* Employees
* Departments
* Jobs
* Locations
* Countries
* Regions
* Dependents

## Analysis Performed

### Employee Analysis

* Highest and lowest salary employees.
* Employee salary classification.
* Employee distribution across departments.

### Department Analysis

* Department-wise employee count.
* Average salary by department.
* High and low performing departments.

### Manager Analysis

* Managers with multiple direct reports.
* Workforce reporting structure analysis.

### Dependent Analysis

* Employees with and without dependents.

---

## Key Findings

* Identified employees with the highest and lowest salaries.
* Analyzed salary distribution across departments.
* Detected employees without dependents.
* Identified managers supervising multiple employees.
* Evaluated workforce distribution across departments.

---

## Business Insights

* Salary differences exist across departments and job roles.
* Some departments appear understaffed.
* Managerial workload varies significantly across teams.
* HR data can support workforce planning and compensation analysis.
